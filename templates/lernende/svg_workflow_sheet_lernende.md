# 🧑‍🎓 SVG Workflow Sheet (für Lernende)

## 🎯 Ziel
Du erstellst eine verständliche Grafik zu einem Informatik-Thema (z. B. Java, Architektur, Ablauf).

Die Grafik soll:
- übersichtlich sein
- fachlich korrekt sein
- leicht verständlich sein

---

## 🧩 1. Wähle den richtigen Grafik-Typ

Was willst du erklären?

- [ ] Vergleich → zwei Dinge gegenüberstellen  
- [ ] Ablauf → Schritt für Schritt zeigen  
- [ ] System → Aufbau eines Systems erklären  
- [ ] Konzept → eine Idee oder ein Prinzip erklären  

---

## ✍️ 2. Inhalte vorbereiten

Bevor du startest, notiere:

### Thema:
```
[Dein Thema]
```

### Lernziel:
```
Was soll jemand nach der Grafik verstehen?
```

---

## ⚙️ 3. Prompt verwenden

### 🔀 Vergleich

```
Thema:
[...]

Linke Seite:
[...]

Rechte Seite:
[...]

Zusammenfassung:
- Punkt 1
- Punkt 2
- Punkt 3
- Punkt 4
```

---

### 🔄 Prozess

```
Thema:
[...]

Schritte:
1. [...]
2. [...]
3. [...]
4. [...]

Hinweis:
[Typischer Fehler oder wichtige Info]
```

---

### 🏗️ Architektur

```
Thema:
[...]

Ebenen:
- Client
- Backend
- Daten

Zusatz:
[Tools oder Systeme]
```

---

### 🧠 Konzept

```
Thema:
[...]

Kern:
[Definition]

Aspekte:
- [...]
- [...]
- [...]

Merksatz:
[Wichtigster Satz]
```

---

## 🎨 4. Regeln für gute Grafiken

- Verwende **kurze Texte**
- Schreibe **klar und einfach**
- Keine langen Sätze
- Alles muss **gut lesbar** sein
- Weniger ist besser als zu viel

---

## 🔍 5. Qualitäts-Check

Kontrolliere:

- [ ] Ist alles verständlich?
- [ ] Ist kein Text abgeschnitten?
- [ ] Ist alles gut lesbar?
- [ ] Sind keine Inhalte doppelt?
- [ ] Ist die Grafik übersichtlich?

---

## 🧪 6. Beispiel

### Thema:
```
Compiler vs Interpreter
```

### Lernziel:
```
Unterschied verstehen
```

### Vergleich:
```
Links:
Compiler → übersetzt Code komplett

Rechts:
Interpreter → führt Code direkt aus
```

---

## 🚀 Tipp

👉 Gute Grafiken erklären ein Thema schneller als Text.

👉 Wenn jemand deine Grafik sofort versteht → Ziel erreicht!
